# C-2 Auth RLS And Security

## Auth and Admin Rules

- Admin APIs/pages require authenticated user.
- User must exist in `admin_users`.
- Guard helpers:
  - `lib/api/guards.ts`
  - `lib/auth/admin.ts`

## RLS Summary

- Public read:
  - `company`
  - `listings` where status is `published`
  - `listing_images` linked to published listings
- Public insert:
  - `inquiries` with listing/company consistency checks
- Admin manage:
  - `company`, `listings`, `listing_images`, `inquiries`, `admin_users`

## Security Helper Function

- SQL function `public.is_admin()` is used by policies.

## Inquiry Abuse Protection

- Honeypot field: `website` must be empty.
- Rate limit window: 10 minutes.
- Limits:
  - max 5 by `ip_hash`
  - max 2 by `ip_hash + listing_id`
- `ip_hash` generated by `sha256(ip + INQUIRY_PEPPER)`.

## Service Role Usage

- Public inquiry endpoint uses service-role client for insert + rate-limit checks.
- `SUPABASE_SERVICE_ROLE_KEY` must remain server-only.
